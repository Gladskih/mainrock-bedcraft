# Security Policy

## Supported Versions

Security fixes are provided for the latest commit on `main`.

## Reporting a Vulnerability

Please do not open a public issue for sensitive findings.

Use one of the following:

1. GitHub Security Advisory ("Report a vulnerability" in the repository Security tab).
2. Private contact with the maintainer, then open a private advisory draft.

Include:

- affected commit/tag
- reproduction steps
- impact assessment
- suggested mitigation (if known)

## Scope

In scope:

- authentication and token cache handling
- LAN discovery and join networking logic
- dependency and supply-chain risks

Out of scope:

- issues caused by unsupported runtime/platform modifications
- vulnerabilities in third-party services outside this repository
